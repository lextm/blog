其实2024年底开始，自己就进入了《.NET 传奇》第二版的写作阶段，也在自己的微信公众号上稍微做了一点点预告。但是2025年间发生了太多计划之外的变故，心情也随着这些起起伏伏，写书的进展就不像第一版那样一气呵成。不过临近年底，终于还是把书的整体框架和大部分章节内容都整理出来了，也顺便孵化了几个有意思的项目，这里就给大家做一下介绍。

## .NET 大事记网站更新

从2016年筹备《.NET 传奇》第一版起，我一直在整理和维护一个名为 [.NET 大事记](https://corefx.lextudio.com) 的网站，专门梳理 .NET 各个版本的发布历史和关键里程碑。这个网站不仅帮助我理清了 .NET 发展的脉络，也自然成为了第一版读者直接可靠的参考资料。不过时过境迁，一是随着 .NET 生态的不断演进，时间线上的内容需要更新和补充，二是当时选择的网页设计过于简单，缺少标记、外链、过滤等方便的交互功能。

根据第二版新章节的实际需要，这次对于每个事件的描述都做了更细致的补充，用标签来展示涉及到的公司、产品（其实包含技术栈、专业领域等等）。当然在大家暂时看不到的幕后，每个事件还附上了对应的参考链接甚至还有一些私密事件的存在，先方便我自己在这个繁忙的写作阶段快速查阅资料，等未来合适的时间再开放给读者。这样全新的需求，也把网站的前端推到了要重新设计的地步，增加了标签外链、按标签过滤等功能，让大家可以更方便地找到自己感兴趣的内容。

随着时间线网站的不断迭代和成熟，我开始在 LinkedIn 上不定期发布“.NET History Quiz”，希望这些生动有趣的形式，让更多人了解 .NET 的演进故事。可以说，这是《.NET 传奇》第二版的“前置工作”，为后续第二版尽快完结打下了坚实的基础。

## 第二版的章节列表

新版《.NET 传奇》预计增加十章新内容，涵盖更多技术细节和生态发展。已经确立大标题的章节包括：

- .NET Core 2.x 和 3.x 的快速成熟之路
- 达成统一与革新的 .NET 5 和其后版本
- Blazor 的全栈野心与挑战
- VS for Mac 的衰落与终结
- Xamarin 的余晖和 .NET MAUI 的反思
- JetBrains Rider 的崛起和完善
- VS Code for C# 的负重前行
- 其他开源 C# IDE 的曲折重生历程
- 其他跨平台 UI 技术的发展与挑战
- 特殊 CPU 架构下的外传：ARM LoongArch RISC-V

大家熟悉的 ILSpy、Avalonia、Uno Platform、OpenSilver 等，也都会在这些章节里以不同的方式出场，讲述它们在 .NET 生态中的角色和影响。相比第一版侧重历史发展脉络，第二版更关注现时的工具和生态视角，力求呈现一个更立体、更完整、也更贴近实际工作的 .NET 世界。这些新增章节不仅讲技术，也讲背后的趣事，帮助读者理解生态间的关联与演变。

## 临时插足的 Project Rover

为了写好 ILSpy 那一章，我重新研究了这个项目的发展历史，以及它曾经迈向跨平台的两次尝试：AvaloniaILSpy 和基于 XPF 的测试版本。当然，写到这两次最终都失败的时候，自己也有点捶胸顿足的遗憾吧。于是就拿着之前评估 Avalonia 11.x 能力时做的某个实验，来试一试能否做出类似 ILSpy WPF 的界面。当然，效果出乎意料的好，也就快速演变成了现在的 Project Rover 预览版。对普通用户来说，它就是一台跨平台的“.NET 程序 X 光机”，在 Windows、macOS、Linux 上都能打开程序集、看看里面的 C# 代码长什么样。

因为自己之前跟 SharpDevelop 项目有过千丝万缕的联系，所以一边做着这个项目一边跟他们团队交流。我们目前的共识是 Project Rover 将暂时按照独立路线图在我的主导下继续进化，而他们继续聚焦在早已确立的 ILSpy v10 开发路线上。两者保持高度同步，但不会互相干扰。

回头看，这次插曲也反过来帮了写书不少忙。一方面，为了不在书里说错话，我不得不把 ILSpy 的演进、WPF/Avalonia 的生态变化梳理得更清楚，很多以前只停留在粗浅印象层面的东西，现在都有了对应的 commit、issue 和实际代码。

另一方面，Project Rover 本身又变成了一份可以公开参考的“样本代码”。如果你手上也有一个 WPF 桌面应用，正在考虑要不要搬到 Avalonia，Rover 肯定可以提供一些有价值的思路，告诉你哪些部分可以最大化复用，哪些部分干脆重写更划算。这方面如果你有兴趣更多了解，欢迎联系 [support@lextudio.com](mailto:support@lextudio.com)。


接下来的一段时间，主线仍然是把书继续写下去。Project Rover 会继续往前走，但节奏会和 ILSpy 主仓库保持同步，也就逐渐缓慢下来，不去抢戏，也当然不会拖后腿。

## 最后

如果你是 ILSpy 的长期用户，欢迎在 Windows 以外的平台上试试 Project Rover 的预览版，看看它能否覆盖你日常用到的那部分需求。如果你是对 .NET 历史感兴趣的读者，不妨把 Rover 当作一份“提前剧透”：它预示着书中相关章节已经从纸面推演，走到了“可以运行的代码阶段”。

等后续章节和项目都有更多进展，再来和大家继续分享这段“写着传奇唱着歌，又搞出了一个工具”的过程。
